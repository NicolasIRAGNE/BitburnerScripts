    WorkloadManager

    A WorkloadManager is responsible for assigning tasks to nodes. It acts as an interface between the user and the nodes.

    (async) assign(task, fill) → {number}

    Assign a task to be executed. The manager will assign the task to multiple nodes if necessary.
    Source:
    • lib/workload_manager.js, line 237
    Parameters:
    Name Type Default Description
    task Task | Batch The task to be executed. Can be a Batch or a Task.
    fill boolean false If true, upon assignment, the entire node RAM will be used to execute the task. This is useful for some misc scripts but it is not recommended for most workloads.
    Returns:
    Type:
    number
    The amount of power that was used to run the task. If this is less than the powerNeeded of the task, the task was not finished. This can also be superior to the powerNeeded of the task if the task does not have the strict_power_allocation flag set.
